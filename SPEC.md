# SPEC.md - AI 狼人杀长期规划与扩展规范

## 1. 文档目的
本文件用于定义项目的中长期架构边界、里程碑和非功能性要求。

使用原则：
- `CLAUDE.md` 关注当前阶段和执行优先级。
- `PROTOCOLS.md` 关注通信格式和动作协议。
- `RULES.md` 关注游戏规则与结算逻辑。
- `AGENTS_SPEC.md` 关注 Agent 推理和行为约束。
- `SPEC.md` 负责把上述文档串成可执行路线图。

## 2. 产品范围
### 2.1 MVP 范围
- 标准 6 人局，后端可完整跑局。
- 支持纯 AI 对局和人机混局基础形态。
- 支持模型不可用时的规则回退策略。
- 支持对局历史查询和基础复盘。

### 2.2 非 MVP 范围
- 语音房、实时语音转写、TTS。
- 匹配系统和账号体系。
- 复杂赛制和观战分房。
- 全量策略学习与强化训练平台。

## 3. 系统架构
### 3.1 模块分层
- `core`: 规则、状态机、结算。
- `agents`: 模型调用、记忆、策略生成。
- `api`: 协议模型与接口入口。
- `frontend`: 展示层和交互层。

### 3.2 关键设计约束
- 状态源唯一：`GameEngine` 为对局真值来源。
- 协议边界清晰：前后端只通过 `schemas` 交互。
- 失败可降级：外部模型和网络波动不阻断对局。
- 可复盘：关键事件必须可追踪。

## 4. 里程碑规划
### M1 - 可稳定跑局 (当前阶段)
- 完成建局、动作提交、自动推进、结束判定。
- 完成最小测试基线（核心逻辑 + API 自动推进）。

### M2 - 人机混局可用
- 增加轮到人类时的等待/超时策略。
- 增加 `pending_actor` 和 `legal_actions` 输出字段。
- 增加前端最小可玩页面。

### M3 - Agent 智能升级
- 落地 system/context/instruction 三层 prompt。
- 引入结构化记忆和检索摘要。
- 增加策略一致性检查（发言与投票一致）。

### M4 - 观战与多房间
- 支持观战视角脱敏推送。
- 增加多房间会话管理和基础并发保护。

## 5. 接口与协议演进策略
- 任何 breaking change 必须先改 `PROTOCOLS.md` 再改代码。
- 新字段默认向后兼容，优先新增可选字段。
- 关键枚举新增后需同步更新：
  - `backend/app/api/schemas.py`
  - `PROTOCOLS.md`
  - 相关测试用例

## 6. 数据与状态策略
- 对局状态当前以内存存储为主，后续可切 Redis。
- 历史事件 `history` 保留关键动作，不暴露不该公开的细节。
- Seer/Werewolf 视角数据必须通过统一遮蔽函数输出。

## 7. 测试策略
- 单元测试：规则结算、阶段转换、胜负判定。
- 接口测试：建局、动作、自动推进、结束状态。
- 回归策略：每次协议变更新增至少 1 条对应测试。

## 8. 运营与观测性要求
- 为接口错误定义稳定错误码（后续迭代）。
- 增加关键指标：单局动作数、回合数、平均耗时、回退触发率。
- 保留最小审计信息，便于排查异常对局。

## 9. 风险与应对
- 模型输出不可控：采用严格 JSON 约束 + 失败回退。
- 规则变更引发回归：通过协议文档驱动测试更新。
- 前后端联调分歧：以 `PROTOCOLS.md` 作为唯一契约。
